# Тестовое задание: Создание REST API приложения

## Описание
Необходимо реализовать REST API приложения для справочника организаций, зданий и деятельностей.

## Предметная область

### 1. Организация
Карточка организации в справочнике содержит:
- Название (например: ООО "Рога и Копыта")
- Номера телефонов (у организации может быть несколько номеров)
- Здание (организация находится в одном конкретном здании)
- Виды деятельности (организация может заниматься несколькими видами деятельностей)

### 2. Здание
Содержит:
- Адрес
- Географические координаты (широта и долгота)

### 3. Деятельность
Позволяет классифицировать род деятельности организаций.
- Имеет название
- Поддерживает древовидную структуру
- Ограничение вложенности: до 3 уровней

Пример дерева:
- Еда
  - Мясная продукция
  - Молочная продукция
- Автомобили
  - Грузовые
  - Легковые
    - Запчасти
    - Аксессуары

### 4. Технологический стек
- FastAPI
- Pydantic
- SQLAlchemy
- Alembic

## Функционал API
Взаимодействие происходит через HTTP-запросы с использованием статического API-ключа.
Все ответы возвращаются в формате JSON.

Реализованные возможности:
- Список всех организаций в конкретном здании
- Список всех организаций по указанному виду деятельности
- Поиск организаций в заданном радиусе или прямоугольной области от указанной точки
- Получение информации об организации по идентификатору
- Поиск организаций по виду деятельности с учетом вложенных дочерних деятельностей
- Поиск организаций по названию
- Ограничение вложенности деятельностей до 3 уровней

## Что нужно реализовать по ТЗ
- Спроектировать БД
- Создать необходимые миграции
- Заполнить БД тестовыми данными
- Реализовать API согласно требованиям
- Упаковать приложение в Docker
- Добавить Swagger UI или ReDoc документацию

## Запуск проекта

### Вариант 1: Docker
1. Перейти в директорию проекта.
2. Запустить:

```bash
docker-compose up --build
```

После запуска доступны:
- Swagger UI: http://localhost:8000/api/docs
- ReDoc: http://localhost:8000/api/redoc

### Вариант 2: Локально
1. Установить зависимости:

```bash
pip install -r requirements.txt
```

2. Создать и заполнить БД (при необходимости):

```bash
python scripts/seed_data.py
```

3. Запустить приложение:

```bash
python run.py
```

## Аутентификация
Для защищенных методов используется статический API-ключ.
Передавайте ключ в заголовке:

- `X-API-Key: <your_api_key>`

Значения ключа и другие настройки задаются через переменные окружения.
